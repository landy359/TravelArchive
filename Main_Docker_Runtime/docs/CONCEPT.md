## 📌 한 줄 요약

봇과의 대화가 지도·달력·일정·메모로 직접 흘러들어가고, 사용자가 그 결과물을 손으로 수정하면 봇이 다시 반응하는 **멀티패널 협업 플래닝 경험**.

---

## 🎯 핵심 시나리오

### 예시 대화
```
사용자: "오사카 3박 4일, 조용한 골목 위주로 잡아줘."

봇:    [지도에 숨은 카페·골목 마커 표시]
       [달력에 3박 4일 여행 기간 설정]
       [일정 패널에 Day 1~4 초안 생성]
       "이렇게 잡아봤어요. 2일차 저녁은 어떻게 할까요?"

사용자: [지도에서 마커 하나를 우클릭해 삭제]
       [달력에서 Day 3 일정을 직접 드래그]
       "마지막 날 오전은 좀 느긋하게 시작하고 싶어."

봇:    [지도 마커·일정 변화를 감지, 컨텍스트 업데이트]
       "알겠어요. 3일차 오전을 자유 시간으로 남겨두고
        숙소 주변 아침 산책 코스 두 곳 추가할게요."
```

## 🎨 멀티패널 구성

### 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│ 좌측 사이드바      중앙 콘텐츠       우측 사이드바       │
│ ──────────────    ──────────────    ────────────────    │
│                                                          │
│ 세션 목록         카카오 지도        달력                │
│ (대화 이력)       (마커 인터랙션)    일정 패널          │
│                   ▲                 메모 패널          │
│ 채팅 창           │ postMessage      ▲                 │
│ (텍스트 입력)     │                 │                 │
│                   ←── 봇 응답 ─────  │                 │
│                                      직접 편집 가능    │
└──────────────────────────────────────────────────────────┘
```

### 각 패널의 역할

| 패널 | 봇이 할 수 있는 것 | 사용자가 할 수 있는 것 |
|:---:|:---|:---|
| 🗺️ 지도 | 마커 추가·제거, 지도 이동 | 좌클릭 마커 추가, 우클릭 제거, 드래그 |
| 📅 달력 | 여행 기간 설정, 날짜에 점 표시 | 날짜 선택, 기간 직접 수정 |
| 📋 일정 | Day별 일정 항목 생성·수정 | 항목 편집, 순서 변경, 삭제 |
| 📝 메모 | 날짜별 요약·메모 작성 | 자유 텍스트 수정 |

## 🔄 데이터 흐름

### Flow 1: 봇 → 패널 (봇이 먼저 반응)

```
사용자 메시지
    ↓
sendMessage (POST /api/sessions/{id}/message)
    ↓
Backend (LLM 처리)
    ├─ map.markers/add     → 지도 마커 추가
    ├─ trip_range PUT      → 달력 기간 설정
    ├─ plan PUT            → 일정 항목 기록
    └─ memo PUT            → 날짜별 메모 기록
         ↓
Parent Window (폴링 or 이벤트)
    ↓
postMessage { type: 'ADD_MARKER', ... }
    ↓
Map iframe (지도에 반영)
```

### Flow 2: 사용자 → 봇 (사람이 먼저 반응)

```
지도 좌클릭 / 우클릭
    ↓
map.js (마커 add/remove)
    ├─ /api/sessions/{id}/map/markers/add      (백엔드 저장)
    ├─ /api/sessions/{id}/map/markers/{id}     (DELETE)
    └─ window.parent.postMessage { type: 'MARKER_ADDED' }
         ↓
Parent Window
    ↓
(선택) 봇에게 "사용자가 마커를 바꿨어요" 컨텍스트 전달
    → sendMessage 또는 자동 요약 트리거
```

## 🤖 향후 확장: 봇 응답 내 구조화 커맨드

봇의 텍스트 응답 안에 패널 조작 명령을 인라인으로 포함시키는 방식. 파싱 레이어가 커맨드를 추출해 각 API를 자동 호출.

### 제안된 JSON 구조

```json
{
  "text": "이렇게 잡아봤어요.",
  "actions": [
    { 
      "type": "ADD_MARKER", 
      "lat": 34.6937, 
      "lng": 135.5023, 
      "title": "도톤보리" 
    },
    { 
      "type": "SET_TRIP_RANGE", 
      "start": "2026-05-10", 
      "end": "2026-05-13" 
    },
    { 
      "type": "ADD_PLAN", 
      "date": "2026-05-10", 
      "items": ["도착", "숙소 체크인"] 
    }
  ]
}
```

**현재 구현 상태**: 텍스트 스트리밍만 지원. 위 구조는 다음 단계에서 설계 예정.

## 💡 왜 "대화를 넘어선 대화"인가?

일반 챗봇은 **텍스트 안에서만** 세계가 완결된다.

이 서비스는 사용자가 **지도를 손으로 만지는 행위 자체가 봇과의 대화**다:

- 마커를 찍으면 "나 여기 가고 싶어"
- 마커를 지우면 "여기는 빼줘"
- 달력에서 날짜를 바꾸면 "일정 조정해줘"
- 메모를 수정하면 "이건 이렇게 해"

봇은 이 신호들을 읽고 플랜을 다시 깎는다. **그게 이 서비스의 핵심 루프다**.
