프로젝트 공식 저장소 (Team Collaboration Hub)

본 저장소는 팀 프로젝트의 메인 소스코드가 관리되는 중앙 저장소입니다.
안정적인 시스템 운영과 원활한 협업을 위해, 모든 팀원께서는 아래의 폴더 구조와 Git 활용 매뉴얼을 반드시 숙지하신 후 개발에 임해 주시기 바랍니다.

[폴더 구조 안내]

1. Main_Docker_Runtime 폴더 (수정 금지)

마스터가 직접 관리하는 핵심 엔진 및 실제 NAS 서버의 Docker 구동 환경입니다.
팀원분들은 본인의 작업물이 메인 시스템과 정상적으로 연동되는지 테스트하기 위한 목적으로만 해당 코드를 다운로드하셔야 합니다. 이 폴더 내부의 파일은 절대 임의로 수정해서는 안 됩니다.


2. Team_Workspace 폴더 (팀원 전용 작업 구역)

팀원분들의 실제 개발이 이루어지는 공간입니다. 부여 받으신 각자의 역할과 기능에 맞추어, 이 폴더 내부에 새로운 파일을 생성하고 코드를 작성해 주시기 바랍니다.
백엔드의 기능은 Main_Docker_Runtime/backend 구조를 참고하여 구현하시기 바랍니다.



---



[팀원 Git 협업 매뉴얼]

Git 사용이 익숙하지 않으신 분들도 아래의 단계별 명령어를 순서대로 입력하시면 안전하게 코드를 관리하실 수 있습니다.


---


0단계: 프로젝트 초기 다운로드 (최초 1회만 실행)

전체 시스템 환경을 개인 PC로 가져오기 위한 과정입니다. 터미널을 열고 아래 명령어를 입력합니다.

```bash
git clone https://github.com/jun-Bridge/TravelArchive.git
cd TravelArchive
```

---


1단계: 개별 작업 공간(Branch) 생성 (새로운 작업을 시작할 때마다 실행)
메인 원본 코드를 안전하게 보호하기 위해, 본인만의 독립된 작업 레이어를 생성해야 합니다.

```bash
git checkout main
git pull origin main
git checkout -b [본인이름_작업내용]
```

(명령어 입력 예시: git checkout -b hong_login_ui)
위 명령어를 모두 입력하신 후, 

```bash
cd Team_Workspace/본인폴더
sudo chmod -R 777 .
```

하고, 내부에서 개발하고자 하는 py들을 작성해주시기 바랍니다.

---


2단계: 작업 내역 서버 업로드 (일일 종료 또는 기능 구현 완료 시 실행)
개별 작업 공간에서 작성한 코드를 GitHub 서버에 안전하게 백업하는 과정입니다.

```bash
git add .
git commit -m "작업 내용을 명확하게 요약하여 작성해 주세요"
git push origin [본인이름_작업내용]
```

---


3단계: 작업물 최종 통합 요청 (Pull Request)
개발이 완료된 코드를 실제 메인 시스템에 반영하기 위해 마스터에게 승인을 요청하는 단계입니다. 
이 과정은 터미널이 아닌 GitHub 웹사이트에서 진행합니다.

1. GitHub의 TravelArchive 저장소 웹페이지에 접속합니다.
2. 화면 상단에 표시되는 Compare & pull request 버튼을 클릭합니다.
3. 작업한 내용을 상세히 기재한 후, 우측 하단의 Create pull request 버튼을 클릭하여 요청을 완료합니다.

이후 마스터가 해당 코드를 검토 및 테스트한 뒤, 이상이 없을 경우 메인 시스템에 최종 반영(Merge)하게 됩니다.

---


4단계: 완료된 작업 공간 삭제 및 초기화 (마스터의 병합 승인 이후 실행)

마스터가 코드를 메인 시스템에 최종 통합(Merge)한 후, 본인 PC에 남아있는 다 쓴 작업 레이어를 깔끔하게 지우고 다음 작업을 준비하는 과정입니다. 이 과정을 거쳐야 파일이 꼬이지 않습니다.

1. 다시 메인 대기실로 이동

```bash
git checkout main
```

2. 마스터가 통합을 완료한 최신 메인 코드로 PC 동기화

```bash
git pull origin main
```

3. 본인 PC에 남아있는 다 쓴 작업 레이어 삭제

```bash
git branch -d [본인이름_작업내용]
```

(명령어 입력 예시: git branch -d hong_login_ui)

여기까지 완료하셨다면, 다음 업무를 배정받았을 때 다시 1단계부터 순서대로 진행하시면 됩니다.
